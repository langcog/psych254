Best Research Practices

  • Once you collect the data, save the "raw" data somewhere safe, and then commit an anonymized version to version control (Git).
    Anonymizing means redacting the workerids (replacing them with numbers/letters) and other identifiable information (IP address, etc)
    Do all analyses on the anonymized data.

  • Document all data transformations using a reproducible script, like an R script. (e.g. converting data from wide form to long form; applying exclusion criteria; etc).
    You should NOT be using Excel to cut and paste to reformat your data. This is NOT reproducible, and is unacceptable for your final project; you should also learn to apply the same mindset to your own research.

Making an external HIT: HTML/Javascript with mTurk

The Computation and Cognition Lab wiki has detailed instructions on how to run external HITs on mTurk. This involves a guide to downloading Amazon's Command Line Tools, writing an experiment, running your experiment, etc. That should be your first stop.

  1. We have a class account for running participants on mTurk.Once you have Amazon's Command Line Tools installed (see Cocolab wiki link above), you will need to modify your mturk.properties file with the access key and password (get this from the TAs).

    This is confidential information which should NOT be put on public version control (i.e. do NOT put your mturk.properties file in a public git repository.) If the credentials gets leaked (e.g. by being on a public repo), then we'll have to re-establish them, making life difficult for many people.

  2. Once you have CLT set up, start making your experiment (see the above wiki for a helpful link, or below for more resources on HTML/Javascript)

  3. Once you have your experiment coded, put it up on the Mechanical Turk Sandbox to test it out. The Sandbox has the same functionality as the "live" mTurk, just that you don't have to pay real money. Because there's no money, there are no "real" participants there, so you needn't worry about people taking your HIT on the Sandbox. The Sandbox is mainly for developers to troubleshoot their HITs.

  4. Get approval from the instructor regarding study design, ethics, power analysis, data analysis plan, etc.
